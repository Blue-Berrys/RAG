## 💡 核心要点

### RAG 三要素
1. **检索**：向量检索、关键词检索、图检索
2. **增强**：查询重写、重排序、父子文档
3. **生成**：格式化生成、Function Calling、流式输出

### 技术演进
```
Naive RAG（基础）
    ↓
Advanced RAG（检索前后优化）
    ↓
Modular RAG（积木式可编排）
    ↓
GraphRAG（知识图谱增强）
```

### 评估体系
- **检索质量**：Context Precision、Context Recall、F1、MRR、MAP
- **生成质量**：Faithfulness、Answer Relevance
- **系统性能**：Latency、Throughput、Cost

## 🎓 面试技巧

### 回答问题的框架
1. **是什么**（What）：定义、概念
2. **为什么**（Why）：原理、优势
3. **怎么做**（How）：实现、应用
4. **对比**（Comparison）：与其他方法对比
5. **实践**（Practice）：项目经验、踩坑

### 展示项目经验
1. **背景**：为什么做这个项目
2. **挑战**：遇到什么问题
3. **方案**：如何解决（技术选型）
4. **效果**：取得了什么成果（数据支撑）
5. **反思**：学到了什么，如何改进
